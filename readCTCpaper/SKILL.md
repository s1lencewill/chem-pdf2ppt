# 理论与计算化学论文阅读笔记生成器

## 核心目标

生成**可直接发布到技术社区**的理论与计算化学论文阅读笔记（知乎、掘金、公众号等）。

笔记要求：
- **内容完整**：理论方法（泛函、基组、力场、ML 势、采样方法等）、计算参数、核心公式不遗漏
- **专业易读**：技术博客风格，既有深度又便于理解，CTC 术语在首次出现时给出简要解释
- **客观准确**：基于论文内容分析，不添加主观臆断
- **深度思考**：通过 Q&A 环节帮助读者深入理解

## 论文类型识别

阅读前先判断论文属于哪种类型，不同类型的笔记侧重不同：

| 类型 | 典型特征 | 笔记重点 |
|------|---------|---------|
| **方法开发** | 新泛函/新力场/新 ML 势/新采样方法 | 设计逻辑、理论推导、基准验证 |
| **方法应用** | 用已有方法研究具体化学问题 | 计算策略、结果解读、机理分析 |
| **benchmark/综述** | 系统性对比多种方法 | 方法间差异原因、适用范围总结 |

## 写作规范

### 应该做

1. **专业准确的表述**
   - 使用计算化学领域规范的术语（泛函名称、基组名称、力场名称、采样方法名等）
   - 公式和符号严格对应论文原文
   - 计算参数和设置描述清晰无歧义
   - 标注方法层级（对 QC：HF → DFT → MP2 → CCSD(T)；对 MD：经典力场 → ML势 → AIMD）

2. **深入浅出的解释**
   - 复杂理论概念先给直觉，再给细节
   - 用类比帮助理解抽象概念（如"metadynamics 像在自由能面上不断填沙子，逼着体系探索未访问区域"）
   - 公式逐项解释变量含义和物理意义

3. **结构清晰的组织**
   - 逻辑层次分明、重点内容突出
   - 适当使用图表辅助说明

4. **有价值的深度分析**
   - 分析方法选择背后的原因（为什么用这个力场/ML 势/采样方法/泛函）
   - 对比与相关工作的方法异同
   - 指出方法的适用范围和局限

### 必须避免

1. **AI 套话和模板句式**
   - "本文的核心贡献是..."、"该方法的优势在于..."、"综上所述..."
   - "值得注意的是..."、"具有重要意义/广泛应用前景..."

2. **空洞的总结和评价**
   - "这是一篇重要的工作"、"为该领域提供了新思路"
   - 不带具体分析的泛泛而谈

3. **过度的格式装饰**：大量 emoji、每句话都加粗、过多层级嵌套

4. **不必要的第一人称**："我认为..."、"我的理解是..."——保持客观叙述视角

## 笔记结构

按以下结构生成笔记。完整模板和示例见 `references/note-template.md`。

### 零、元信息（笔记开头）

```markdown
> **论文标题**
> **作者**
> **发表**
> **难度** ⭐~⭐⭐⭐⭐⭐
> **前置知识** [列出需要的基础概念]
```

**难度等级说明**：
- ⭐ 入门级：只需基础化学知识
- ⭐⭐ 基础级：了解计算化学基础概念（如势能面、力场）
- ⭐⭐⭐ 进阶级：熟悉 DFT、MD 等常用方法
- ⭐⭐⭐⭐ 专业级：需要较深的增强采样/ML 势/耦合簇等领域知识
- ⭐⭐⭐⭐⭐ 专家级：涉及复杂理论推导或前沿方法开发

### 一、总览
#### 创新点
用 2-3 句话概括论文的核心创新，不超过 100 字。
必须包含：问题背景 + 核心方案（含方法层级）+ 关键创新点。
#### 摘要
提取摘要要点罗列出来
### 二、论文概述

- **解决什么问题**：一句话描述
- **核心方案**：一句话概括方法类型（如"NNP + metadynamics 计算界面酸碱性"）
- **主要贡献**：2-3 点列举

### 三、背景与动机

- 现有方法怎么做的，存在什么问题/瓶颈
- 问题的根本原因（精度？计算量？无法描述特定效应？）

### 四、核心方法

根据论文类型，有侧重地展开以下内容：

#### 整体计算策略

- 给出计算流程架构（如：DFT 参考数据 → 训练 ML 势 → MD 采样 → 自由能分析）
- 说明数据流向和多尺度衔接关系

#### 势能面描述（必选其一）

**量子化学方法**（静态计算、反应机理类论文）：
- 理论方法层级、泛函选择、基组、色散校正、溶剂化模型
- BSSE 校正、相对论效应等
- 详细参考 `references/ctc-methods.md`

**经典力场**（生物/材料 MD 类论文）：
- 力场名称与版本（AMBER、CHARMM、OPLS、COMPASS 等）
- 水模型（TIP3P、SPC/E、TIP4P 等）
- 特殊参数化（配体、金属中心等是否自定义参数）

**机器学习势函数**（MLP/NNP 类论文）：
- ML 架构（DeepMD、GAP、SchNet、NequIP、MACE 等）
- 训练数据来源和生成策略（参考级别、主动学习/DP-GEN 等）
- 描述符/特征类型
- 训练精度和验证方式

**第一性原理 MD**（AIMD/CPMD/BOMD 类论文）：
- MD 类型（Born-Oppenheimer MD / Car-Parrinello MD）
- 电子结构设置（泛函、基组/平面波截断能）
- 与经典 MD/ML 势 MD 的成本-精度取舍

#### 动力学与采样（如涉及）

- **MD 类型**：经典 MD / AIMD / ML 势 MD / QM/MM-MD
- **系综**：NVE / NVT / NPT，温控/压控方式（Nosé-Hoover、Langevin、Berendsen 等）
- **时间步长与模拟时长**
- **增强采样**（如涉及）：metadynamics、umbrella sampling、REMD、STE-MD、OPES 等
  - 集体变量（CV）的选择及理由
  - 偏置势参数（Gaussian 高度/宽度、沉积频率等）
  - 自由能重构方法
- **自由能计算**：TI、FEP、WHAM、MFI 等
- 详细参考 `references/md-methods.md`

#### 分析工具与参数

- 使用的软件及版本（GROMACS、CP2K、LAMMPS、DeepMD-kit、PLUMED 等）
- 关键计算参数（截断半径、静电处理方法、周期性边界条件等）
- 轨迹分析方法

### 五、结果与讨论

#### 思路整理
1. 先计算/模拟了什么？获得了什么信息？
2. 之后得出什么结论？接下来算什么？
3. 各部分结果之间的逻辑串联

#### 结果详情
1. 每个关键结果的物理/化学含义，按逻辑演进
2. 图表的具体内容和结论
3. 结果对解决核心问题的作用

### 六、总结与思考

```markdown
## 总结

### 核心贡献
[具体列出论文的理论/方法/应用贡献]

### 局限性
[方法限制、适用范围、计算成本、假设条件等]

### 适用场景
[该方法/结论适合/不适合哪些体系或问题]
```

## Q&A 环节

生成 3-5 个深度 Q&A，覆盖以下类型：

1. **原理理解类**：为什么选这个 CV？为什么这个 ML 势架构而不是另一个？
2. **细节辨析类**：某个参数/操作的具体含义；易混淆的方法区分（如 umbrella sampling vs metadynamics 的适用场景差异）
3. **边界条件类**：什么体系/条件下方法会失效？假设条件是什么？
4. **延伸思考类**：能否迁移到其他体系？有哪些可能的改进方向？

答案要求：直接展示、有理有据、适当举例、对未说明的部分标注为"推测"。
