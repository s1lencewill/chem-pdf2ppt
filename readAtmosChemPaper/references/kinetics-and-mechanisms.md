# 动力学、机制和模型分析清单

## 动力学参数

常见参数：

- 速率常数 `k`，常见单位为 `cm³ molecule⁻¹ s⁻¹` 或 `s⁻¹`。
- 摄取系数 `γ`，用于异相/多相反应。
- 产率 `Y`，如 SOA yield、molar yield、mass yield。
- 分支比 branching ratio。
- 光解频率 `J`。

检查项：

- 参数是直接测量、相对速率法、绝对速率法、拟合还是模型反演。
- 温度、湿度、压力、NOx、粒径、相态等依赖是否给出。
- 是否与 IUPAC、JPL、MCM 或前人推荐值对比。
- 是否报告误差范围和适用条件。

## 反应机制分析

机制部分必须拆成以下层次：

1. 初始反应：OH/NO3/O3/Cl/光解等如何启动。
2. 中间体：RO2、RO、过氧化物、羰基、硝酸酯、有机酸等。
3. 分支路径：与 NO、HO2、RO2、O2、H2O、表面/液相组分的竞争。
4. 终产物：气相产物、低挥发性产物、SOA 组分。
5. 观测解释：机制解释了哪些时间序列、质谱峰、产率变化或环境依赖。

写作要求：

- 对每个关键步骤标注证据来源：实验观测、标准品确认、模型拟合、文献类比或作者推测。
- 如果存在替代机制，说明作者如何排除或为什么未能排除。
- 不把分子式指认直接写成唯一结构。

## 产物与中间体证据分级

如果论文给出反应机制图或机制路径，必须把图中关键物种拆出来逐一判断证据状态。机制图不是证据本身，它通常混合了直接检测物种、推断中间体和文献类比路径。

### 必须回答的问题

- 该物种是否被本文实验检测到？
- 如果检测到，用的是什么手段：CIMS、PTR-MS、GC-MS、LC-MS、AMS、FTIR、UV-Vis、NMR、MS/MS、标准品、色谱保留时间、同位素标记等。
- 检测证据能支持 `m/z`、分子式、官能团类别，还是具体结构？
- 如果没有检测到，作者为什么把它放进机制：模型拟合、产物关联、已知自由基化学、MCM/IUPAC/JPL 机制、前人文献还是相似体系类比？
- 参考文献支持的是同一个体系，还是相似反应/相似官能团/通用速率参数？

### 结构确定程度

| 等级 | 含义 | 常见证据 |
|---|---|---|
| L0 | 未检测，只是机制假设或可能路径 | 机制图、作者推测、模型需要 |
| L1 | 检测到特征信号但不能给唯一分子式 | 低分辨 `m/z`、AMS 碎片峰、宽峰 |
| L2 | 支持分子式但不能确认唯一结构 | HRMS 精确质量、同位素峰 |
| L3 | 支持结构类别或官能团 | MS/MS 碎片、衍生化、FTIR 官能团、反应性测试 |
| L4 | 支持具体结构 | 标准品、色谱保留时间、多仪器交叉验证 |
| L5 | 强结构确认 | NMR、晶体结构或等效强证据 |

### 参考文献追踪

需要查阅以下参考文献：

- 机制段落直接引用的文献。
- 产物指认、速率常数、摄取系数、分支比引用的文献。
- 机制模型来源，如 MCM、IUPAC、JPL、AtChem、F0AM 或作者新增反应的来源。
- 支持相似体系类比的文献。

写作时说明参考文献支持的具体点：

| 参考文献 | 支持内容 | 与本文关系 | 限制 |
|---|---|---|---|
| Ref. X | 某产物曾在相似 VOC 氧化中被检测 | 相似体系类比 | 不能证明本文体系中该物种存在 |

避免把“前人观察到相似产物”写成“本文直接检测到该产物”。

## 模型和前人机制对比

常见模型：

- MCM：近显式 VOC 氧化机制。
- F0AM、AtChem：0-D 箱模型。
- PyCHAM：烟雾箱和气溶胶过程模拟。

检查项：

- 模型的输入：初始浓度、光解频率、稀释、壁损、温湿度。
- 模型的机制来源：MCM、IUPAC、JPL、作者新增反应。
- 是否做敏感性分析。
- 模型是否能同时解释反应物衰减、产物生成和气溶胶演化。
- 作者新增机制是否只是改善拟合，还是有独立实验支持。

## 与前人相比的突破分类

将“突破”归入具体类别：

| 类型 | 判断标准 |
|---|---|
| 新产物 | 识别了前人未检测或未确认的产物 |
| 新路径 | 提出并支持新的反应通道或分支 |
| 新参数 | 给出新的速率常数、摄取系数、产率或分支比 |
| 新条件 | 覆盖前人缺失的 RH、NOx、温度、粒径、相态等条件 |
| 新仪器证据 | 用更高分辨率、更低检出限或结构确认方法解决歧义 |
| 模型修正 | 修改 MCM/JPL/IUPAC/箱模型机制并改善实验解释 |
| 不确定度收窄 | 更可靠地约束已有参数或机制 |

避免写“首次”“突破性”等强表述，除非原文和证据明确支持。
