# 大气化学实验论文阅读笔记生成器

## 核心目标

生成**可直接发布到技术社区**的大气化学/气溶胶实验论文阅读笔记。笔记必须回答四个问题：

1. 作者为什么研究这个物质、反应或体系？
2. 实验实际测到了什么，证据是否可靠？
3. 作者提出的反应机制解释了什么现象？
4. 和前人相比，本文的突破、修正或增量在哪里？

笔记要求：

- **证据链清楚**：关键结论必须绑定原文位置，如 Section、Figure、Table、Page；必要时给短引文或精确转述。
- **实验细节完整**：反应器、条件、仪器、校准、空白、壁损、传输损失、不确定度不遗漏。
- **机制分析克制**：区分直接观测、拟合结果、模型推断和作者假设，不把推测写成事实。
- **物种证据分级**：机制图中的产物和中间体必须逐一判断是实验检测、间接推断还是文献/机理假设。
- **图表可复用**：提取关键图片，转写关键表格，用图表解释机制、实验设计和核心结论。
- **专业易读**：面向大气化学、气溶胶和环境化学读者，首次出现的专业术语给出简短解释。

## 论文类型识别

阅读前先判断论文属于哪些类型，可多选：

| 类型 | 典型特征 | 笔记重点 |
|---|---|---|
| 烟雾箱实验 | 室内光化学/暗反应，控制 VOC、NOx、RH、种子气溶胶 | 条件代表性、壁损校正、SOA 产率、产物时间演化 |
| 流动管/反应管实验 | 短停留时间、动力学或异相反应 | 反应时间、传输损失、摄取系数、速率常数 |
| 外场观测 | 实地采样、多仪器联合观测 | 来源解析、相关性、环境条件、统计稳健性 |
| 仪器方法开发 | 新检测方法、新校准、新解析流程 | 灵敏度、检出限、干扰、验证实验 |
| 动力学/机理实验 | 速率常数、产物分支、自由基机制 | 反应路径、拟合方法、机制证据 |
| 气溶胶物性实验 | 粒径、吸湿性、CCN、挥发性、相态 | 物性参数、气粒分配、气候意义 |
| 实验-模型联合 | MCM/F0AM/AtChem/PyCHAM/箱模型 | 实验约束、机制缺口、敏感性分析 |

## 本地执行与容错流程

### Windows/PowerShell 编码

读取中文 skill、PDF 抽取文本或写出中文 Markdown 时，显式使用 UTF-8：

```powershell
Get-Content -LiteralPath '<path>' -Encoding UTF8
$env:PYTHONIOENCODING='utf-8'
```

如果终端显示乱码，先确认源文件是否真有编码问题；不要把 PDF 抽取文本中的乱码复制进笔记。

## 材料读取总流程

按以下顺序处理材料，避免把“正文抽取”和“图表获取”割裂开：

1. **材料盘点**：自动扫描当前目录和一层子目录内的 PDF、DOCX、XLSX、CSV、ZIP、PNG/JPG/TIF 等文件，判断主文、SI、source data、官方图和其它参考材料。
2. **DOI/网页核对**：如果主文首页、文件名或正文含 DOI，打开 DOI/期刊官方页面，核对题名、作者、期刊、年份，并查找 Figures、Supplementary Information、Source Data、Code/Data availability。
3. **文本抽取**：对主文、SI、DOCX 等材料分别抽取文本，保存为 `*.extracted.txt`。
4. **图表获取**：优先使用官方图、source data、本地 SI 图；若不可用，再用 PDF 渲染裁剪或内嵌图片提取。
5. **证据来源标注**：写 README 时标注每条关键证据来自主文、SI、source data、DOI 页面还是参考文献。

### 文件自动识别规则

推荐命名只是建议，不是要求。必须先自动扫描文件，再综合判断材料角色。

扫描范围：

```text
当前目录
当前目录的一层子目录
```

扫描类型：

```text
*.pdf
*.docx
*.xlsx
*.csv
*.zip
*.png / *.jpg / *.jpeg / *.tif / *.tiff
```

角色判断：

| 角色 | 判断依据 |
|---|---|
| 主文 | 文件名含 `article`、`paper`、`main`、`manuscript`、`正文`；首页含 DOI、标题、作者、Abstract、Introduction；正文含 References；文件名不含 SI 关键词 |
| SI / Supplementary | 文件名含 `supp`、`supplementary`、`SI`、`ESM`、`MOESM`、`supporting`、`补充`；首页含 `Supplementary Information` 或 `Supporting Information`；正文含 `Supplementary Fig.`、`Supplementary Table`、`Supplementary Methods` |
| Source Data | 文件名含 `source data`、`sourcedata`、`figshare`、`dataset`、`raw`、`data`；Excel/CSV sheet 或列名含 `Fig.`、`Figure`、`Supplementary Fig.`、`Table` |
| 官方图/图片 | 文件名含 `fig`、`figure`、`scheme`、`mechanism`、`graphical`，或位于 `figures/`、`images/` 目录 |
| 其它参考材料 | 与主文 DOI/标题相关，但不满足以上类别 |

如果文件名相似但角色不清楚，不要只凭文件名判断；必须结合首页文本、DOI/标题匹配、Supplementary Fig/Table、References、页数和文件类型。若仍无法确定，在 README 的“已核查材料”表中标注“角色推断”。

建议在 README 末尾加入材料识别表：

```markdown
| 文件 | 角色判断 | 判断依据 | 是否读取 | 备注 |
|---|---|---|---|---|
| xxx.pdf | 主文 | 首页含 DOI/Abstract/References | 是 |  |
| xxx_ESM.pdf | SI | 首页含 Supplementary Information 和 Supplementary Fig. | 是 |  |
| source_data.xlsx | Source Data | sheet 名含 Fig. 1 | 是/否 |  |
```

### 文本抽取流程

解析 PDF 正文时按以下顺序尝试：

1. 优先使用 `pdftotext` 等 PDF 文本提取工具。
2. 若不可用，fallback 到 Python `pypdf` 提取页面文本。
3. 将主文保存为 `paper.extracted.txt`；将 SI 保存为 `supplementary.extracted.txt`；其它材料用原文件名派生 `*.extracted.txt`。
4. 若某段文本抽取失败，只基于可验证的 PDF 页面、DOI 页面、图注、表格或上下文校正；不要猜测缺失数值。

DOCX/XLSX/CSV 处理：

- DOCX：抽取标题、正文、表格和图片说明。
- XLSX/CSV：优先读取 sheet 名、列名、图表数据和 table 数据；不要把未读取的 source data 写成已复核。

### 输出产物 checklist

每篇论文默认生成以下产物：

- `paper-notes/<slug>/README.md`：主阅读笔记。
- `paper-notes/<slug>/paper.pdf`：原始论文 PDF 副本。
- `paper-notes/<slug>/paper.extracted.txt`：PDF 正文抽取中间文件。
- `paper-notes/<slug>/images/`：关键图片和表格截图。
- 可选 `paper-notes/<slug>/supplementary.*`：Supplementary Information 副本。
- 可选 `paper-notes/<slug>/source-data/`：Source Data、Excel、CSV、网页下载图表数据。

### 硬性完成定义与验收命令

除非用户明确要求“只要摘要”或“不要插图”，否则本 skill 的任务不能只生成文字摘要。完成前必须满足：

1. **材料读取完成**：主文和可识别的 SI 必须抽取文本并保存为 `*.extracted.txt`；若抽取失败，必须在 README 和最终回复中说明原因。
2. **关键图表插入完成**：README 至少插入主文核心图表，优先 Fig. 1、Fig. 2、Fig. 3 及机制/模型/实验设计图；若论文没有这些图号，则选择支撑核心结论的关键图。图像必须保存到 `images/` 并用 Markdown 图片语法插入。
3. **图注双语化完成**：每张关键图的图片下方必须给出“图题概括”“原文图注（规范化转写）”和“中文概要”。不能只保留 `Fig. X | title`，也不能只写“图显示了 XXX”。
4. **化学式和单位规范化完成**：README 正文中的化学式、离子、电荷和单位必须用可复制的 Unicode 上下标/上标，避免把正文化学式写成会渲染为斜体的 LaTeX 行内公式。
5. **证据边界完成**：关键结论表、仪器证据表、产物/中间体证据分级表和已核查材料表必须存在。
6. **最终检查完成**：发送最终回复前必须运行下面的检查命令；对不合理残留必须修正，合理残留要能解释。

```powershell
rg -n "NO2-|NO3-|NOx|RO2|HO2|H2O2|O3|M\^-|s\^-|m\^-3|cm\^-3|ng m\^-3|ug m\^-3|μg m\^-3|\$[A-Za-z0-9_\\^{} +.-]+\$" paper-notes/<slug>/README.md
rg -n "!\[|原文图注|中文概要|已核查材料|结构确定程度|证据边界" paper-notes/<slug>/README.md
rg -n "图题概括|原文图注（规范化转写）|机制速览|→|⇌" paper-notes/<slug>/README.md
Get-ChildItem paper-notes/<slug>/images
```

不得提前结束的情况：

- README 没有插入任何图片。
- 没有生成主文 `paper.extracted.txt`。
- 没有做化学式/单位规范化检查。
- 引用了 Supplementary Figure/Table 但 SI 未读取或未在“已核查材料”中说明。
- 缺少 PyMuPDF/pypdf/pillow 等工具时直接跳过图表获取；应先尝试官方图/source data，再尝试安装或说明安装失败。

### 推荐输入文件约定

这些命名可以加速识别，但不是硬性要求：

| 类型 | 推荐命名示例 |
|---|---|
| 主文 | `paper.pdf`、`article.pdf`、`main.pdf`、`manuscript.pdf` |
| SI | `supplementary.pdf`、`si.pdf`、`supporting_information.pdf`、`*_ESM.pdf`、`*_MOESM*.pdf` |
| Source Data | `source_data.xlsx`、`source-data/`、`figshare/`、`dataset.xlsx` |
| 官方图片 | `figures/`、`images/`、`figure1.png`、`scheme.png` |

即使文件未按推荐命名，也必须通过自动识别规则分类。

```text
│   ├── README.md

│   ├── supplementary.pdf

│       ├── fig2_result_time_series.png
│       ├── fig3_mechanism_scheme.png
│       └── table1_key_conditions.png
```

### 图表获取优先级

对主文和 SI 图表按以下顺序处理：

1. 本地 source data、官方图片或 `figures/` 目录。
2. DOI/期刊官方页面可下载的 Figures、Supplementary Information、Source Data。
3. 本地 SI PDF/DOCX 中的图表。
4. PyMuPDF 渲染 PDF 页面并裁剪关键图。
5. `pypdf` 的 `page.images` 提取 PDF 内嵌位图。

原则：

- 官方 DOI 页面和本地 SI/source data 的优先级高于 PDF 截图。
- 若 SI 未能读取，不得把 Supplementary Figure/Table 的内容写成“已核查”；只能写“主文提及，SI 未核查”。
- 凡是关键结论或产物证据依赖 Supplementary Figure/Table，必须在“已核查材料”表和对应证据表中标注 SI 是否已读。

### 推荐工具

推荐环境：

```powershell
pip install pypdf pymupdf pillow
```

用途：

- `pypdf`：正文抽取、页数和元信息读取、少量内嵌图片提取。
- `pymupdf`/`fitz`：PDF 页面渲染、关键图表裁剪、矢量图处理 fallback。
- `pillow`：裁剪、保存和检查图片尺寸。

如果 PyMuPDF 未安装：

1. 先尝试 DOI 官方图和本地 SI/source data。
2. 若必须裁剪 PDF，再按环境许可安装到项目局部依赖目录。
3. 任务完成后清理临时依赖目录，保留最终图片。

## 笔记结构

按以下结构生成笔记。完整模板见 `references/note-template.md`。

### 零、元信息

```markdown
> **论文标题**
> **作者**
> **发表**
> **研究对象**
> **实验类型**
> **同行评议状态/版本**
> **难度** ⭐~⭐⭐⭐⭐⭐
> **前置知识**
```

### 一、为什么研究这个物质/体系

必须回答：

- 研究对象是什么：物质、反应、气溶胶体系或环境过程。
- 它从哪里来：天然源、人为源、二次生成、燃烧、生物源、海洋源等。
- 为什么重要：大气寿命、氧化能力、SOA/O3 生成、气候效应、健康效应、政策或观测意义。
- 前人不知道什么：产物不清楚、速率不确定、机制有争议、环境条件缺失、仪器无法区分等。

### 二、论文概述

- **解决什么问题**：一句话描述。
- **核心实验方案**：一句话概括实验设计和测量链路。
- **主要贡献**：2-3 点，必须具体。
- **证据类型**：直接观测、校准实验、对照实验、拟合、箱模型、机理推断。

### 三、实验设计与测量链路

根据论文内容展开：

- **反应器/采样场景**：烟雾箱、流动管、反应管、外场站点、气液界面装置等。
- **反应条件**：温度、相对湿度、压力、光源/光强、NOx/VOC、O3、OH/NO3/Cl 来源、种子气溶胶、停留时间。
- **测量仪器**：CIMS、PTR-MS、GC-MS、AMS、SMPS、CPC、CCN、FTIR、UV-Vis 等。
- **校准与质量控制**：标准物、内标、空白、背景扣除、检出限、灵敏度、重复性。
- **校正项**：壁损、传输损失、稀释、湿度影响、离子化效率、气粒分配、采样效率。

实验方法细节参考 `references/atmos-exp-methods.md` 和 `references/instruments-and-uncertainty.md`。

### 四、实验仪器速读：这些仪器到底证明了什么

面向理论计算或非实验背景读者，必须把关键仪器翻译成“它约束了机制中的哪一环”。每种仪器按以下问题解释：

- 缩写、中文名和英文全称分别是什么？
- 它测什么？
- 本文用它回答什么问题？
- 它能证明什么？
- 它不能证明什么？
- 证据强度对应 L0-L5 或高/中/低的哪一级？

必须使用 9 列表格：

```markdown
| 缩写 | 中文名 | 英文全称 | 简单理解 | 本文用途 | 能证明什么 | 不能证明什么 | 本文对应证据 | 证据强度 |
|---|---|---|---|---|---|---|---|---|
| IC | 离子色谱 | Ion Chromatography | 分离并定量水溶液中的目标无机离子 | 跟踪 NO₂⁻ 消耗和 NO₃⁻ 生成 | 能直接定量无机离子变化 | 不能识别 ON 结构 | Fig. X | 强：对无机离子定量强；对 ON 结构无直接证据 |
```

要求：

- 不能只写 IC、CIMS、NMR 等缩写；必须写中文名和英文全称。
- “简单理解”要让非实验背景读者能理解。
- “能证明什么”和“不能证明什么”都必须填写，防止把仪器信号过度解释成机制事实。
- “证据强度”不能只写“强/中/弱”，必须说明强在哪里、弱在哪里，并尽量对应 L0-L5。
- 模型方法也要进入表格，如 RACM2/箱模型、ISORROPIA-II、PMF、MCM、F0AM 等。
- 至少解释实际出现在论文里的关键仪器/模型。

### 五、关键图表和表格

优先提取或转写以下内容，并用“图表解读卡片”写成可直接用于 PPT 汇报的结构化说明：

- 实验装置图、反应流程图、采样路径图。
- 时间序列图：反应物、氧化剂、产物、粒径、质量浓度、信号强度。
- 质谱图：CIMS/PTR-MS/AMS/GC-MS 的峰指认和产物证据。
- 粒径分布、SOA 产率、O:C/H:C、氧化态、吸湿性、CCN 活性。
- 机制图和模型-实验对比图。
- 实验条件表、主要产物表、速率常数/摄取系数/产率表、仪器参数表、不确定度表。

- **机制图**：说明每条箭头代表的反应步骤；标注哪些物种直接观测、哪些是推断。
- **质谱图**：说明 `m/z`、离子式、峰归属、对照组和结构确认等级。
- **时间序列图**：说明横轴、纵轴、反应条件、对照组和反应组的变化趋势。
- **产率/速率图**：说明坐标、分组、关键数值、误差棒含义和参数化关系。
- **模型图**：说明输入参数、情景、颜色/符号、坐标、主要结果和模型边界。
- **表格**：转写关键数值，并说明这些数值进入了哪条结论或模型计算。

表格既可截图保存，也要尽量转写成 Markdown 表格，便于检索和复用。

图注要求：

- 每张插入 README 的关键图，必须紧跟“图题概括”“原文图注（规范化转写）”和“中文概要”。
- 原文图注优先来自 PDF/HTML 正文 caption；不要用自己改写的英文冒充原文。
- “原文图注（规范化转写）”不能只保留 `Fig. X | title`。如果原文 caption 包含 panel、条件、颜色/符号、误差棒或 Source Data 信息，必须一并转写。
- 原文图注过长时可摘录关键句，但必须保留图号、panel、条件、变量、误差棒和 Source Data 等关键信息。中文概要可完整转述。
- 图题概括、原文图注（规范化转写）、中文概要和正文机制链都必须使用 Unicode 化学式/单位；不要在这些位置使用 LaTeX 行内公式。
- 如果图注来自 SI、source data 或 DOI 页面，必须在“图表来源”和“已核查材料”中标注。

图表提取细节参考 `references/note-template.md`。

图表来源优先级：

- 官方网页下载图或 source data 图。
- 本地 SI 或 source data 中的原图/表格。
- PyMuPDF 渲染 PDF 后裁剪。
- `pypdf` 提取 PDF 内嵌图片。

### 六、反应机制：解释了什么

如果论文涉及反应机制，必须先给出 1 个“机制速览：一眼看懂的反应链”，用 Unicode 化学式和箭头表示核心路径。示例：

```text
NO₂⁻ + H⁺ ⇌ HONO
ROOH + HONO → ROONO + H₂O
ROONO → [RO• ·NO₂]
[RO• ·NO₂] → RONO₂
```

要求：

- 使用 Unicode 上下标/电荷，不能用 `NO2-`、`H2O` 或 LaTeX 行内公式。
- 只写核心路径，不把所有副反应塞进速览。
- 机制速览后续必须逐步说明每一步的证据级别：直接观测、间接推断、文献类比或作者假设。
- 如果机制存在竞争路径，另列“竞争/副路径”。

不要只复述机制图，必须回答：

- 机制解释了哪些观测现象：产物来源、时间演化、SOA 增强/降低、O3 生成、NOx/RH/光照依赖、异相摄取、气粒分配等。
- 哪些步骤有直接实验依据，哪些来自模型或作者推断。
- 关键中间体、自由基链传递、产物分支和终止路径是什么。
- 是否存在竞争机制或替代解释。

动力学和机制分析参考 `references/kinetics-and-mechanisms.md`。

### 七、产物与中间体证据分级

如果论文给出实验反应机制，必须逐一检查机制图、正文机制段落、产物表和补充信息中的产物/中间体。不要默认机制图中的物种都被实验检测到。

必须回答：

- 该物种是**直接检测到**、**间接推断**、还是**机制假设**？
- 如果检测到，用了什么手段：CIMS、PTR-MS、GC-MS、LC-MS、AMS、FTIR、UV-Vis、NMR、MS/MS、标准品、色谱保留时间、同位素实验等。
- 实验证据能支持到什么程度：只有 `m/z`、分子式、官能团/结构类别，还是具体结构？
- 如果是推测，推测依据是什么：实验趋势、碎片峰、产物关联、箱模型拟合、MCM/IUPAC/JPL 机制、相似体系类比、前人文献。
- 作者引用了哪些参考文献来支持该中间体、产物、速率常数、分支比或反应路径？需要查阅这些参考文献的题名、结论和它们支持的具体点。

必须生成固定表格：

```markdown
| 物种/中间体 | 在机制中的角色 | 证据状态 | 检测/推断手段 | 结构确定程度 | 推测依据 | 参考文献依据 | 可信度 |
|---|---|---|---|---|---|---|---|
| XXX | 中间体/产物/自由基 | 直接检测/间接推断/机制假设 | CIMS m/z、GC-MS、MS/MS、标准品、模型拟合等 | L0-L5 | 实验趋势、碎片峰、MCM、类似体系 | Ref. 12, Ref. 35 | 高/中/低 |
```

结构确定程度分级：

- **L0**：未检测，只是机制假设或作者提出的可能路径。
- **L1**：检测到 `m/z`、特征信号或低分辨峰，但不能给出唯一分子式。
- **L2**：高分辨质谱或同位素信息支持分子式，但不能确认唯一结构。
- **L3**：MS/MS、碎片、官能团证据或衍生化实验支持结构类别。
- **L4**：标准品、色谱保留时间、多仪器交叉验证支持具体结构。
- **L5**：NMR、晶体结构或等效强结构证据确认具体结构。

参考文献使用规则：

- 优先查机制段落、产物指认段落、速率常数/分支比段落中直接引用的文献。
- 如果论文声称“consistent with previous studies”或“based on known chemistry”，必须找到对应参考文献并说明它支持的是产物、路径、速率还是类比。
- 如果无法获取参考文献全文，至少使用题名、摘要、DOI 页面或可访问片段判断其支持范围，并标注证据限制。
- 不要把参考文献支持的“相似体系”写成对本文体系的直接证明。

### 八、与前人相比的突破

必须单独成节，避免空泛评价：

- 前人怎么解释或测量？
- 前人缺少什么条件、产物、仪器证据、动力学参数或模型约束？
- 本文新增了什么：新产物、新路径、新速率、新条件、新仪器分辨能力、新模型修正。
- 本文是机制突破、数据补充、条件扩展，还是不确定性的收窄？

### 九、关键结论与原文依据

必须生成固定表格：

```markdown
| 关键结论 | 原文依据 | 位置 | 证据类型 | 可信度 |
|---|---|---|---|---|
| 中文转述，不夸大 | 短引文或精确转述 | Section/Figure/Table/Page | 实测/拟合/模型/推测 | 高/中/低 |
```

规则：

- 关键结论不能只写“作者认为”，必须指向原文证据。
- 如果是短引文，每条只摘必要短句；更长内容用中文转述。
- 发现证据不足时标注“证据不足”或“推测”。
- 若依据来自 SI，必须写明 Supplementary Figure/Table 编号，并确认 SI 已读取；若未读取，标注“主文提及，SI 未核查”。

### 十、局限性和环境代表性

重点检查：

- 实验条件是否接近真实大气：浓度、湿度、NOx、光照、停留时间、种子气溶胶。
- 反应器壁损、传输损失、仪器干扰是否可能改变结论。
- 产物识别是否存在同分异构体、碎片峰、加合离子或交叉灵敏度问题。
- 箱模型是否缺少关键反应，参数是否来自推荐值或拟合。
- 外推到真实大气时哪些结论可靠，哪些需要谨慎。

## Q&A 环节

生成 3-5 个深度 Q&A，覆盖以下类型：

1. **研究动机类**：为什么这个物质或反应值得研究？
2. **实验设计类**：为什么使用这个反应器、氧化剂来源、湿度/NOx 条件？

## 写作规范

### 应该做

- 使用大气化学领域规范术语：SOA、VOC、NOx、RO2、HO2、OH、NO3、Cl、uptake coefficient、rate constant、branching ratio 等。
- **正文化学式优先使用 Unicode 上下标/上标**，不要默认使用 LaTeX 行内公式。目标是可直接复制且不被渲染成斜体，例如 `NO₂⁻`、`NO₃⁻`、`H₂O₂`、`N₂O₅`、`RO₂`、`HO₂`、`NOₓ`、`O₃`、`C₄H₁₀NO₃⁺`、`[RO• •NO₂]`。
- **LaTeX 只用于真正数学公式、复杂推导或 Markdown 渲染必须保留的公式**，不要把普通正文化学式写成 `$NO_2^-$`、`$H_2O_2$` 这类会斜体化的形式。
- 单位写清楚并优先使用 Unicode 指数，例如 `cm³ molecule⁻¹ s⁻¹`、`M⁻¹ s⁻¹`、`s⁻¹`、`ng m⁻³ h⁻¹`、`μg m⁻³`、`ppbv`、`pptv`。
- PDF 抽取文本中的 ASCII 化学式必须后处理：`NO2- → NO₂⁻`，`NO3- → NO₃⁻`，`H2O2 → H₂O₂`，`NOx → NOₓ`，`RO2 → RO₂`，`HO2 → HO₂`，`O3 → O₃`，`M^-1 s^-1 → M⁻¹ s⁻¹`，`ng m^-3 h^-1 → ng m⁻³ h⁻¹`。
- 明确区分“观测到”“拟合得到”“模型预测”“作者推测”。
- 图表解释要具体到变量、趋势、条件和结论。
- 对机制图中的每个关键物种标注结构确定等级 L0-L5。
- 在 README 末尾列出“已核查材料”，说明主文、SI、source data、DOI 页面是否读取。

### 必须避免

- 空泛评价：“具有重要意义”“提供了新思路”“有广阔前景”。
- 只复述摘要，不分析实验可靠性。
- 把模型输出当作实验直接证据。
- 把机制图中的中间体默认写成“已检测到”。
- 把高分辨质谱分子式默认写成“结构已确认”。
- 把相似体系文献支持写成本文体系的直接证明。
- 在没有读取 SI 的情况下，把 Supplementary Figure/Table 写成已核查证据。
- 从低分辨率图中强行估读精确数值。
- 忽略校准、空白、壁损、传输损失和不确定度。
- 在正文中大量使用 LaTeX 行内化学式导致普通化学式不可直接复制或呈现为斜体。
